# ADR 0004: Uniqueness for CFIs — `(tenant_id, opd_invoice_id)` is the business key

- **Status:** Accepted
- **Section in brief:** 7.3a

## Context

The natural business invariant is: one OPD invoice maps to exactly one Consultation Fees Invoice. The CFI's `opd_invoice_id` should be unique per tenant. But the worker also uses `event_id` for idempotency ([[0003-idempotency|ADR 0003]]). The two keys serve different purposes.

## Options considered

- **(a) Two unique constraints: `event_id` (UUID, generated by HMS) and `(tenant_id, opd_invoice_id)` (business key)** — the worker idempotency uses `event_id`; the business invariant is also enforced at the DB level.
- **(b) Only `event_id`** — the business invariant is enforced at the application layer (the worker checks `opd_invoice_id` before inserting).

## Decision

**(a) Both constraints.** `event_id UNIQUE` for worker idempotency, `(tenant_id, opd_invoice_id) UNIQUE` as the business invariant.

## Rationale

- Defense in depth: a future code path that bypasses the worker (e.g., a manual fix-up script) still cannot create two CFIs for the same OPD invoice.
- The two constraints catch different bugs: `event_id` catches worker retry bugs; `(tenant_id, opd_invoice_id)` catches logical bugs in any code that creates CFIs.
- Cost: two indexes on a single table that will be small (< 1M rows in v1). Negligible.

## Consequences

- Schema: `UNIQUE (tenant_id, opd_invoice_id)` and `UNIQUE (event_id)` on `consultation_fees_invoices`.
- The worker's idempotency check uses `event_id` (the `ON CONFLICT (event_id) DO NOTHING`).
- The business invariant check (also in the worker, as a sanity check) is `SELECT 1 FROM consultation_fees_invoices WHERE tenant_id = $1 AND opd_invoice_id = $2` before insert. If a row exists with a different `event_id`, the worker logs an error and pauses the outbox row for manual review — this would indicate a real bug (the same OPD invoice triggered two different events).
- A manual data-fix script must drop the `(tenant_id, opd_invoice_id)` constraint explicitly to run. That's intentional.
