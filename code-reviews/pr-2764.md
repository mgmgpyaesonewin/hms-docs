## H1. `deleteServiceWithDependencies` silently loses the foreign-key 409 guard for services with `labGroup` history

**Where:** `src/app/(dashboard)/shared/opd/repositories/service.repository.ts:312-323`, `src/app/(dashboard)/shared/opd/services/service.service.ts:212`

The old `deleteService` wrapped `prisma.service.delete` in a try/catch that converted any FK constraint violation into a friendly 409:

```ts
// service.repository.ts:292-310 (pre-PR)
async deleteService(id: string) {
  try {
    return await prisma.service.delete({ where: { id } });
  } catch (error) {
    if (isForeignKeyConstraintViolationError(error)) {
      throw new AppError(
        "This service has previous history.You cannot delete right now.",
        409,
      );
    }
    throw error;
  }
}
```

The PR replaces the call site with a new method that does **not** apply that guard:

```ts
// service.repository.ts:312-323 (post-PR)
async deleteServiceWithDependencies(id: string) {
  return await prisma.$transaction(async (tx) => {
    await tx.labTest.deleteMany({ where: { serviceId: id } });
    return tx.service.delete({ where: { id } });
  });
}
```

The migration `hms-app/prisma/migrations/20260312170714_add_lab_group_table_and_modified_related_table/migration.sql` defines `lab_groups.labTestId` as `ON DELETE RESTRICT`. So `tx.labTest.deleteMany` itself throws P2003 (FK violation) whenever a `labGroup` row references this `labTest` — i.e. for *any service that has ever been used in a lab order*, which is exactly the "previous history" case the old guard was designed to communicate.

Because the new method does not catch and translate the error, the user now sees a raw Prisma error string instead of the friendly 409 message `"This service has previous history.You cannot delete right now."`. The PR improves one case (brand-new service with no `labGroup`) at the cost of regressing every other case.

**Fix:** keep the friendly 409 path; only delete `labTest` when it is safe:

```ts
async deleteServiceWithDependencies(id: string) {
  return await prisma.$transaction(async (tx) => {
    const labTest = await tx.labTest.findFirst({
      where: { serviceId: id },
      select: {
        id: true,
        _count: { select: { labTestMappings: true } },
      },
    });

    if (labTest && labTest._count.labTestMappings > 0) {
      throw new AppError(
        "This service has previous history. You cannot delete right now.",
        409,
      );
    }

    if (labTest) {
      await tx.labTest.delete({ where: { id: labTest.id } });
    }

    return tx.service.delete({ where: { id } });
  });
}
```

(`lab_template_items` is `ON DELETE SET NULL` so it does not block deletion; only `lab_groups` does.)
