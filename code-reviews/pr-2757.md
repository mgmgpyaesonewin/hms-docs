## H1. Orphan `Tabs.Tab value="CathLab Services"` left in `Tabs.List` — clicking it renders nothing

**Where:** `hms-app/src/app/(dashboard)/emr/cathlab/features/components/cathlab-ipd-emr-tabs-component.tsx:109`

The PR removes the `<Tabs.Panel value="CathLab Services">…</Tabs.Panel>` (lines 191-207 in the pre-merge file) but leaves the matching tab trigger in `Tabs.List`:

```tsx
<Tabs.List my={"md"} style={{ flexWrap: "nowrap", overflowX: "auto", overflowY: "hidden" }}>
  …
  <Tabs.Tab value="CathLab Services">CathLab Services</Tabs.Tab>
  <Tabs.Tab value="Service Request">Service Request</Tabs.Tab>
  …
</Tabs.List>
```

After the deletion, clicking "CathLab Services" produces an empty tab — Mantine `Tabs` with no matching `Tabs.Panel` for the active value renders a blank content area. The tab button still has the `CathLab Services` permission subject, so `PermissionGuard` never gets a chance to block access (the guard was inside the deleted panel), meaning **any user who can see the cathlab EMR tabs sees a clickable, blank "CathLab Services" tab** that does nothing.

Worse, line 196 of `cathlab-services/cathlab-ipd-emr-services-tab-component.tsx` (`router.push(\`/cathlab/emr-creation-cathlab/${resultIpdEmrId}/edit?tab=CathLab Services\`)`) still navigates users to that URL — the EMR creation flow now lands the operator on a dead tab after saving.

**Fix:** remove the `Tabs.Tab` trigger as part of this PR (or, if the intent is to keep the entry but route it to the `IpdEmrServiceRequestList` panel, rename the tab value and add a redirect). Same applies to the `?tab=CathLab Services` link in the services component.
